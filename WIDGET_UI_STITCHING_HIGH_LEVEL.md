# Widget UI Stitching - High-Level Implementation Document

Generated from current repository state on 2026-04-15.

## 1) Scope

This document covers only the web widget stitching work:

- JS embed snippet loader
- Hosted widget app (`/chat?embed=1`)
- Admin Step 3 configuration UI (colors, size, position, channels, preview, embed code)
- Integration points with existing chat/admin APIs

No backend business logic or chat pipeline behavior is changed by this stitching.

## 2) Tech Stack Used

### Frontend

- React 18 + TypeScript
- Vite 5 multi-entry build (`index.html` and `chat.html`)
- Tailwind CSS
- Framer Motion (widget transitions)
- Lucide icons
- Sonner toast

### Embed Layer

- Vanilla JavaScript loader script (`kebo-widget-loader.js`)
- Iframe-based embedding into third-party websites
- `window.postMessage` bridge for open/close/toggle

### Backend (existing, reused)

- FastAPI (Python)
- Existing admin APIs under `/admin/api/*`
- Existing chat APIs under `/api/chat/*`
- Static hosting for loader via `/static`
- Chat host route via `/chat`

## 3) What Is Implemented

### A) Admin Step 3 now drives full widget config

File: [admin_ui/src/components/bot-training/ChannelsStep.tsx](/new_bot/admin_ui/src/components/bot-training/ChannelsStep.tsx)

- Brand color inputs for all 4 fields:
  - `primaryColor`
  - `accentColor`
  - `bgColor`
  - `textColor`
- Widget controls:
  - position (`left` or `right`)
  - width
  - height
- Channel toggles:
  - Web widget enabled
  - WhatsApp enabled
- Embed snippet generation includes all color fields and size/position fields
- Copy-to-clipboard button for snippet
- Live widget preview shown on the right side of brand colors section

### B) Step 3 config is loaded and saved with existing endpoints

File: [admin_ui/src/pages/app/BotTraining.tsx](/new_bot/admin_ui/src/pages/app/BotTraining.tsx)

- Default channel/theme values are defined in `defaultChannels`
- UI settings are read from `/admin/api/config/onboarding/ui`
- Business channels fallback is read from `/admin/api/config/onboarding/business`
- Save action writes to `/admin/api/config/onboarding/ui` with:
  - `theme.primary_color`, `theme.accent_color`, `theme.background_color`, `theme.text_color`
  - `widget.position`, `widget.width`, `widget.height`
  - `channels.web_widget.enabled`, `channels.whatsapp.enabled`

### C) Embeddable JS loader script is implemented

File: [static/embed/kebo-widget-loader.js](/new_bot/static/embed/kebo-widget-loader.js)

- Reads `data-*` attributes from snippet
- Normalizes values (numbers, booleans, colors, position)
- Creates floating fixed root container in website bottom left/right
- Injects iframe pointing to `/chat?embed=1` with query params
- Supports open/close/toggle state with launcher button
- Exposes public API:
  - `window.KeboWidget.open()`
  - `window.KeboWidget.close()`
  - `window.KeboWidget.toggle()`
  - `window.KeboWidget.destroy()`
- Handles viewport resize and mobile keyboard offsets

### D) Hosted widget runtime is implemented in React chat page

File: [admin_ui/src/pages/ChatHarness.tsx](/new_bot/admin_ui/src/pages/ChatHarness.tsx)

- Reads embed query params into runtime config:
  - `brand_color`, `accent_color`, `bg_color`, `text_color`
  - `position`, `width`, `height`
  - `widget_id`, `hotel_code`, `phase`, `session_id`, `bot_name`
- Applies all 4 colors across widget surfaces
- Embed mode behavior:
  - Header fixed at top of widget
  - Footer input area fixed at bottom
  - Only message list (`overflow-y-auto`) scrolls
- Non-embed mode still supports launcher/minimize/expand behavior

### E) Backend serving wiring for widget host

Files:
- [main.py](/new_bot/main.py)
- [api/routes/admin.py](/new_bot/api/routes/admin.py)

- `/chat` serves built `admin_ui/dist/chat.html`
- `/static` serves loader file from `static/embed/kebo-widget-loader.js`
- `/admin` continues serving admin SPA and existing admin APIs

## 4) End-to-End Flow

1. Admin updates Step 3 settings and clicks Save.
2. Frontend saves theme/widget/channels to `/admin/api/config/onboarding/ui`.
3. Embed code block updates with current values.
4. External website includes the snippet.
5. Loader script mounts floating launcher and iframe.
6. Iframe opens `/chat?embed=1...` with all runtime parameters.
7. ChatHarness runs in embed mode and calls existing chat APIs.

## 5) Embed Snippet Contract

Current snippet generated by Step 3:

```html
<script src="http://<host>/static/embed/kebo-widget-loader.js"
  data-widget-id="<property_code>"
  data-hotel-code="<property_code>"
  data-phase="pre_booking"
  data-brand-color="#xxxxxx"
  data-accent-color="#xxxxxx"
  data-bg-color="#xxxxxx"
  data-text-color="#xxxxxx"
  data-bot-name="Nova"
  data-position="right"
  data-width="380"
  data-height="600">
</script>
```

Optional supported attributes:

- `data-host`
- `data-session-id`
- `data-api-base`
- `data-auto-open`
- `data-offset`
- `data-z-index`

## 6) UI Behavior Fixes Included

- Widget remains docked to configured side (`left` or `right`) in floating mode.
- Header and footer are non-scrolling sections.
- Message thread is the only scrolling section.
- All 4 admin color inputs are respected by embed runtime and preview.
- Step 3 preview is placed beside Brand Colors for cleaner configuration UX.

## 8) Run and Verify

1. Build frontend:
   - `cd admin_ui`
   - `npm run build`
2. Run backend:
   - `python main.py`
3. Open admin:
   - `http://localhost:8000/admin`
4. In Setup Wizard Step 3:
   - change colors and dimensions
   - save
   - copy embed code
5. Paste snippet before `</body>` in test website HTML and load page.
6. Confirm:
   - widget launches bottom side as configured
   - theme colors match all 4 fields
   - header/footer stay fixed while messages scroll
